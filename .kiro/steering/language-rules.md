# 语言规则

## 文档语言要求

所有规范文档（specs）必须使用**中文**编写，包括：

- 需求文档 (requirements.md)
- 设计文档 (design.md)
- 任务文档 (tasks.md)

## 具体要求

1. **用户故事**：使用中文描述
2. **验收标准**：使用中文描述，但保留 EARS 模式关键词（WHEN、THE、SHALL、IF、THEN、WHERE、WHILE）
3. **术语表**：中文名称 + 英文标识符（用于代码引用）
4. **设计文档**：使用中文描述架构、组件和接口
5. **任务文档**：使用中文描述实现步骤

## 代码相关内容

以下内容保持英文：
- 代码示例和代码块
- 文件名和路径
- 技术术语的英文标识符
- API 名称和接口定义
